# Data Model: Todo Full-Stack Web Application - Phase II

## Task Entity

### Attributes
- **id**: Integer
  - Type: int
  - Description: Primary key, auto-incremented unique identifier
  - Constraints: Primary key, auto-increment
  - Default: Auto-generated

- **user_id**: String
  - Type: str
  - Description: Foreign key linking to user who owns the task
  - Constraints: Not null, references users.id (managed by Better Auth)
  - Default: Set during creation

- **title**: String
  - Type: str
  - Description: Title of the task
  - Constraints: Not null, 1-200 characters
  - Default: ""

- **description**: String
  - Type: str
  - Description: Detailed description of the task
  - Constraints: Optional, max 1000 characters
  - Default: ""

- **completed**: Boolean
  - Type: bool
  - Description: Indicates whether the task is completed
  - Constraints: Boolean value
  - Default: False

- **created_at**: Timestamp
  - Type: datetime
  - Description: When the task was created
  - Constraints: Not null
  - Default: Current timestamp

- **updated_at**: Timestamp
  - Type: datetime
  - Description: When the task was last updated
  - Constraints: Not null
  - Default: Current timestamp, updated on modification

### Relationships
- **Belongs to**: User (user_id â†’ users.id, managed by Better Auth)
- A Task belongs to exactly one User
- A User can have many Tasks

### Validation Rules
- title must be 1-200 characters
- description must be max 1000 characters
- user_id must reference a valid user
- completed defaults to False

## User Entity (Managed by Better Auth)

### Attributes
- **id**: String
  - Type: str
  - Description: Primary key, unique identifier for user
  - Constraints: Primary key
  - Default: Auto-generated by Better Auth

- **email**: String
  - Type: str
  - Description: User's email address
  - Constraints: Unique, not null
  - Default: Set during registration

- **name**: String
  - Type: str
  - Description: User's display name
  - Constraints: Optional
  - Default: ""

- **created_at**: Timestamp
  - Type: datetime
  - Description: When the user account was created
  - Constraints: Not null
  - Default: Current timestamp

### Relationships
- **Has many**: Tasks (via tasks.user_id)
- A User can have many Tasks
- A Task belongs to exactly one User

## API Endpoints Data Flow

### Request/Response Objects

#### Task Request Object
- **title**: String (required, 1-200 chars)
- **description**: String (optional, max 1000 chars)

#### Task Response Object
- **id**: Integer
- **user_id**: String
- **title**: String
- **description**: String
- **completed**: Boolean
- **created_at**: String (ISO format)
- **updated_at**: String (ISO format)

#### Authentication Request Object
- **email**: String (required)
- **password**: String (required)
- **name**: String (required for signup)

#### Authentication Response Object
- **token**: String (JWT token)
- **user**: Object (user details)
